# Deploy Web Catalog (GitHub Pages)

## Architecture

**Realtime Mode (DEFAULT)**
- Web viewer fetches extension data directly from remote sources in real-time
- No rebuild needed (data is always fresh)
- Sources configured in `web/remote-sources.json` (auto-synced from `references/remote-sources.json`)

**Snapshot Mode (FALLBACK)**
- Static cached files: `plugin.json`, `catalog.json`, `site-health.json`
- Only used when realtime fetch fails (e.g., network issues)
- Must be regenerated by `npm run build:catalog` when personal extensions change

## Deployment Pipeline

Workflow: `.github/workflows/deploy-pages.yml`

Triggered on push to `main` branch containing changes in `web/**`.

## Pre-deployment checklist

### If you changed personal extensions (extensions/ folder):
```bash
npm run build:catalog
git add extensions/ web/
git commit -m "chore: rebuild extension catalogs"
git push origin main
```

### If you changed source list (references/remote-sources.json):
```bash
npm run sync:web-catalog
git add web/remote-sources.json
git commit -m "chore: sync remote sources to web"
git push origin main
```

### If you changed BOTH:
```bash
npm run full-sync
npm run sync:health:all
git add extensions/ web/
git commit -m "chore: full sync - catalogs and sources"
git push origin main
```

### If you want web viewer to regenerate snapshots without rebuilding extensions:
```bash
npm run sync:web-catalog
```

This will regenerate:
- `web/plugin.json` (snapshot fallback)
- `web/catalog.json` (snapshot fallback sidecar)
- `web/remote-sources.json` (realtime source list)

## Health check commands

```bash
# Default legacy alias (web plugin source URLs)
npm run sync:web-health

# Explicit scopes
npm run sync:health:web
npm run sync:health:realtime
npm run sync:health:all
```

Khuyen nghi truoc khi deploy: `npm run sync:health:all` de check ca URL site ext va URL repo realtime.

## Local Testing

```bash
cd web
python -m http.server 8000
# Then open: http://localhost:8000/

# Test realtime mode (default):
# http://localhost:8000/?realtime=1

# Test snapshot mode (fallback):
# http://localhost:8000/?catalog=1
```

## What gets deployed

- `web/index.html` - UI
- `web/script.js` - Rendering logic  
- `web/data.js` - Realtime/snapshot loader
- `web/style.css` - Styles
- `web/theme.js` - Theme toggle
- `web/plugin.json` - Snapshot fallback (auto-generated)
- `web/catalog.json` - Snapshot fallback (auto-generated)
- `web/remote-sources.json` - Realtime config (auto-generated)
- `web/site-health.json` - Health monitoring (auto-generated)
- `web/.nojekyll` - GitHub Pages config
- `web/DEPLOY.md` - This file (reference)
python -m http.server 8000 --directory web
```
